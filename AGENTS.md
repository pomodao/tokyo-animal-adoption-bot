# AGENTS.md

## 目的

このリポジトリは、東京都動物愛護相談センターの譲渡猫ページを低コストで定期監視し、新しい掲載があったときに SNS へ投稿する bot を実装・運用するためのものです。

## プロダクト目標

- 固定費をできるだけ抑える
- 意味のある更新を安定して検知する
- 重複投稿を防ぐ
- 運用負荷を低く保つ
- 監視対象サイトに過度な負荷をかけない

## 作業方針

- 常時稼働サーバーよりも、無料または低コストで維持できるシンプルな定期実行基盤を優先する
- 投稿先の主対象は `Bluesky` とする
- `X` は API コストの観点から任意機能として扱う
- 更新検知は生の HTML 全体ではなく、正規化した動物レコードで行う
- 動物エントリの識別子は `管理番号` を主に使う
- 要件が広がるまでは、まず「新規掲載」の通知のみを対象にする
- 監視間隔は保守的に設定し、通常は `2〜3時間` とする
- 初期構成では `GitHub Actions` を定期実行基盤として採用する
- 状態は外部ストアではなくリポジトリ内の管理用ファイルで保持する
- 可能な範囲で掲載画像も取得し、画像付きで投稿する
- public repository の scheduled workflow が 60 日非アクティブで無効化されうることを前提に設計する
- 60 日対策として、事前警告用 Issue の作成と、必要時の手動再有効化を運用に含める

## ドキュメント方針

- リポジトリ内の運用・設計ドキュメントは原則として日本語で記述する
- アーキテクチャ上の意思決定は `docs/` 配下に記録する
- 次の変更が入ったときは関連ドキュメントを更新する
  - ホスティング基盤
  - 状態保存方式
  - 監視間隔
  - 対応 SNS
  - 実装言語方針
  - 差分判定ルール

## 実装ガイドライン

- HTML の小さな変更に耐えられるパーサーを優先する
- リトライや再デプロイ後でも重複投稿しないだけの状態を保持する
- パース結果に確信が持てない場合は推測で投稿しない
- 監視状態は git 管理されるファイルとして保存し、変更履歴を追えるようにする
- 画像 URL や画像取得結果も状態として扱い、同じ画像の重複投稿を避ける
- state ファイルは毎回更新せず、掲載内容や投稿結果に意味のある差分があるときだけ更新する
- 失敗監視と通知は state ファイルではなく GitHub Actions の run 履歴と Issue で扱う
- 各 TypeScript ファイルには、そのファイルの責務や意図が分かる最小限の日本語コメントを入れる
- 各関数には JSDoc 形式で、その関数が何をするかを日本語で記述する
- TypeScript の型だけで十分に意図が伝わる場合は `@param` などを省略し、型だけでは伝わりにくい前提や意味がある場合だけ補足する
- 実装は次の 3 つの責務に分離する
  - 取得・パース
  - 差分判定・状態管理
  - SNS 投稿

## 運用ガイドライン

- 監視対象サイトへのアクセス頻度を抑え、可能ならランタイムで識別可能な User-Agent を設定する
- 取得、パース、差分判定、投稿の結果を追跡できるログを残す
- 取得失敗やパース失敗が続いた場合に検知できるようにする
- 認証情報は必ずプラットフォームの Secrets に保存し、リポジトリに含めない
- scheduled workflow の無効化を避けるため、最終コミットからの日数を定期的に監視する
- 無効化が近いと判断したら管理者向け Issue を作成して対応を促す
- workflow が無効化された場合は GitHub UI から再有効化して運用を継続する
